# Солнце Москвы — Выводы и рекомендации

> [!info] Как читать этот документ
> Рекомендации отсортированы по приоритету. Всё, что отмечено как «✅ оба» — подтверждено двумя независимыми исследованиями. Это самые надёжные рекомендации. Вещи, отмеченные одним источником, помечены соответственно.

---

## Стратегические выводы

### 1. Два бренда в одном (✅ оба)

Сайт обслуживает два принципиально разных сценария:
- **«Хочу покататься на колесе»** — аттракцион, билеты, виды
- **«Хочу провести время в комплексе»** — еда, развлечения, шопинг, прогулка

Сейчас приоритет отдан колесу, МФК на вторых ролях. Но МФК — это 26 000 кв.м, ~37 арендаторов, и отдельный бизнес, который генерит трафик вне зависимости от колеса. Редизайн должен дать обоим направлениям равноценное представление.

### 2. Мероприятия — главная точка роста маржи (✅ оба)

10+ пакетов мероприятий (свадьбы, корпоративы, свидания, дни рождения) — это направление с самым высоким средним чеком. Но раздел «Организовать праздник» слабо упакован: нет галерей, нет цен, нет портфолио, нет быстрой формы.

> [!tip] Бенчмарк
> London Eye выделяет Private Pod как полноценный продукт с отдельной страницей, FAQ, add-on'ами и fast-track. Это модель для «Солнца Москвы».

### 3. Разорванный UX покупки (✅ оба)

Переход на tickets.moscow-sun.ru = переход на «другой сайт» (другой дизайн, другая навигация, другой логотип). Это ломает путь пользователя и снижает доверие. Нужна либо бесшовная интеграция (виджет/iframe), либо минимум визуальное единство.

### 4. Несогласованность «операционных истин» (✅ оба)

> [!danger] Самая опасная проблема
> На сайте минимум **4 разных версии графика работы** в разных разделах. Это прямое топливо для негативных отзывов и обращений в поддержку. Решается не дизайном, а моделью данных (единый источник → трансляция везде).

Расхождения обнаружены в: часах работы, тарифах, правилах (аттракцион vs МФК), контактах.

### 5. Навигация от структуры организации, а не от задач пользователя (✅ оба)

Критичные страницы (цены, очередь, как добраться, правила) спрятаны в футере. Пользователь видит 4–5 пунктов в шапке, а нужная информация — в нескольких кликах. Для такого объёма контента нужно либо мегаменю, либо сильные landing-страницы категорий.

## Что исправить — Приоритет 1: КРИТИЧНО (неделя 1)

> [!danger] Без этого редизайн будет косметикой

### 1.1 Единый источник данных (✅ оба)

- [ ] Единый компонент для часов работы → трансляция на все страницы
- [ ] Единый блок тарифов → синхронизация с билетным сервисом
- [ ] Единый блок ограничений (погода, животные, возраст)
- [ ] Механизм обновления цен «за минуты» (повышение с 1 мая)

### 1.2 Убрать мёртвый контент (✅ оба)

- [ ] Страница «Ковид-19» в футере билетного сервиса — удалить
- [ ] Устаревшие карты в сайтмапе (active-map_old, active-map-3)
- [ ] PHP-файлы в сайтмапе (32MKBbLaQLzgwfmJkn3TSbe.php)
- [ ] Личные страницы в индексе (/personal/*, /auth/)

### 1.3 SEO-основа (по данным SEO-аудита, балл 34/100)

- [ ] Добавить meta description на все страницы (сейчас ноль)
- [ ] Добавить тег H1 на каждую страницу (сейчас нет)
- [ ] Добавить JSON-LD разметку: LocalBusiness + TouristAttraction
- [ ] Добавить hreflang теги для 4 языковых версий (RU, EN, CN, AE)
- [ ] Добавить lang="ru" в HTML
- [ ] Почистить sitemap.xml

### 1.4 Статус работы колеса (✅ оба)

- [ ] Реалтайм-виджет или заметный блок: «Колесо работает / не работает сейчас»
- [ ] Привязка к погодным данным или хотя бы телефон для уточнения

## Что исправить — Приоритет 2: ВАЖНО (недели 2–3)

### 2.1 Навигация (✅ оба)

- [ ] Мегаменю или расширенная навигация вместо 4–5 пунктов
- [ ] Вынести в основную навигацию: Билеты, Планируйте визит, Мероприятия
- [ ] FAQ — из футера в основное меню
- [ ] CTA «Купить билет» — всегда видна (sticky)

### 2.2 Новые ключевые страницы (✅ оба)

- [ ] **«Планируйте визит» / Plan Your Visit** — объединить: как добраться, парковка, часы работы, правила, что взять, советы, доступность
- [ ] **«О комплексе»** — история, архитектура Chapman Taylor, факты, видео, галерея
- [ ] **Парковка** — отдельный блок (сейчас отсутствует полностью)
- [ ] **Комбо-билеты** — с ВДНХ, Москвариумом, Останкинской башней

### 2.3 Упаковка мероприятий (✅ оба)

- [ ] Каждый тип мероприятия = отдельная продуктовая страница
- [ ] Галерея, цена/от, что входит, ограничения, FAQ
- [ ] Форма заявки с согласием на ПДн
- [ ] Портфолио / фото прошлых мероприятий

### 2.4 Социальное доказательство (✅ оба)

- [ ] Интегрировать реальные отзывы на сайт (виджет Яндекс/2ГИС или подборка)
- [ ] Рейтинги на видном месте (5.0 Яндекс, 4.8 2ГИС)
- [ ] UGC-контент (фото посетителей)
- [ ] Блок «СМИ о нас» — перенести на основной сайт

### 2.5 Бесшовность покупки (✅ оба)

- [ ] Единый визуальный язык: основной сайт ↔ билетный сервис
- [ ] Одинаковые названия типов билетов
- [ ] Согласованная навигация
- [ ] Логотип — устранить различия (на билетном сервисе другой)

## Что исправить — Приоритет 3: РЕКОМЕНДАЦИИ (месяц 1–2)

### 3.1 Контент и SEO

- [ ] Запустить блог / раздел статей (целевые запросы: «колесо обозрения Москва», «что посмотреть ВДНХ»)
- [ ] Страница «Что видно с колеса» — интерактивная панорама или фотогид
- [ ] Разделить «Акции» / «События» / «Новости» как разные типы контента
- [ ] FAQ с Schema-разметкой (FAQPage)
- [ ] Open Graph теги для красивых превью в соцсетях
- [ ] Файл llms.txt для AI-поисковиков

### 3.2 Мультиязычность

- [ ] EN-версия основного сайта (минимум) — сейчас EN есть только на билетном сервисе
- [ ] CN-версия (опционально, но ВДНХ посещают китайские туристы)
- [ ] Согласовать языковые версии между сайтом и билетным сервисом

### 3.3 Интерактивная карта МФК (✅ оба)

- [ ] Полная переработка (по ТЗ клиента)
- [ ] Текстовый слой со ссылками для SEO и доступности (карта/фильтры плохо индексируются)
- [ ] Стандартизированные карточки арендаторов (название, описание, этаж, часы, кухня, средний чек, фото)

### 3.4 Управление ожиданиями (✅ оба)

- [ ] Страница «Советы перед посещением» — когда ехать, что взять, как экономить
- [ ] Честное предупреждение о фото на входе (от 800 ₽)
- [ ] Объяснение ценности прозрачного пола (для кого, что даёт)
- [ ] Сравнение с другими смотровыми площадками (помогает обосновать цену)

### 3.5 Подарочные сертификаты (Claude)

- [ ] Отдельная страница — популярная услуга для колёс обозрения в мире
- [ ] London Eye, Singapore Flyer — все продают сертификаты онлайн

## Что исправить — Приоритет 4: НА БУДУЩЕЕ

### 4.1 Технические улучшения

- [ ] Dynamic pricing (цена по дате/времени/спросу, как London Eye)
- [ ] Сезонные опыты внутри кабинок (рождественский, весенний)
- [ ] 360°-тур или видео из кабинки для страницы «Колесо обозрения»
- [ ] Программа лояльности (или интеграция с «Друзья ВДНХ»)
- [ ] Мобильное приложение (отложить на второй этап, как в брифе)

### 4.2 B2B-блок

- [ ] Корпоративным клиентам — выделенная страница
- [ ] Калькулятор стоимости мероприятия
- [ ] Портфолио корпоративных мероприятий

## Структурные UX-проблемы (сводка)

| Проблема | Источник | Влияние | Решение |
|----------|----------|---------|---------|
| Раздробленная навигация | ✅ Оба | Потеря пользователей | Мегаменю + landing-категории |
| Билеты на поддомене | ✅ Оба | Разрыв UX, потеря доверия | Визуальное единство / виджет |
| Несогласованность часов | ✅ Оба | Негативные отзывы | Единый источник данных |
| Нет «О комплексе» | ✅ Оба | Нет имиджевой страницы | Новый раздел |
| Карта спрятана | ✅ Оба | 37 арендаторов невидимы | Вывести в основную навигацию |
| Нет блока отзывов | ✅ Оба | Потеря социального доказательства | Виджет + рейтинги |
| Нет статуса погоды | ✅ Оба | Люди приезжают зря | Реалтайм-виджет |
| Акции = Афиша = Новости | ✅ Оба | Путаница контента | Разделить на 3 типа |

## Технические рекомендации (сводка)

| Область | Что сделать | Приоритет |
|---------|------------|-----------|
| SEO | Meta description, H1, JSON-LD, hreflang, sitemap | Критично |
| Производительность | Убрать дубли аналитики, оптимизировать видео | Важно |

## Метрика «Первый удар по пользе» (✅ оба)

> [!important] Финальный вывод
> Оба исследования сходятся в одном: в редизайне надо **не добавлять красивые страницы, а убрать неопределённость**: цены / часы / очередь / ограничения / как добраться / как купить безопасно и без посредников. Именно на этом «ломаются» отзывы, именно это перехватывают конкуренты и агрегаторы.
